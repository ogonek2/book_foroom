1. Короткий підсумок

За результатами сесії доопрацювань платформа отримала повноцінний інструментарій для наповнення каталогу, зручніший імпорт, вкладені категорії та покращений досвід користувача на сайті.

Ключові досягнення

  • Імпорт книг — оновлений формат Excel, розділення авторів на імʼя/прізвище, підтримка підкатегорій, нові поля (цикл, формат, збірник).
  • Адмін-панель — кілька авторів на книгу, довідник форматів видання, налаштовувані заглушки обкладинок, вкладені категорії.
  • Сайт — сторінка каталогу категорій, модальне вікно фільтрації, повний український алфавіт авторів.
  • Надійність — виправлення років видання при імпорті, синхронізація звʼязку автор–книга, кеш і відновлення імпорту Gutenberg.
Увага (відновлення пароля): листи скидання пароля надсилаються через SMTP. На сервері потрібно перевірити налаштування MAIL_* у файлі .env. Без коректного пароля поштової скриньки функція не працюватиме (замість помилки 500 користувач тепер бачить зрозуміле повідомлення).

2. Заглушки обкладинок та фото авторів

Окремі зображення-замінники для книг і авторів, що зберігаються локально та керуються з адмінки.

НовеАдмінка

Що зроблено

  • Сторінка в адмін-панелі: Заглушки зображень (/admin11/image-placeholders).
  • Окремі заглушки для обкладинки книги та фото автора.
  • Файли зберігаються в public/images/placeholders/, шлях — у таблиці site_settings.
  • На всьому сайті (Blade, Vue) використовуються поля cover_image_display та photo_display з автоматичним fallback.

3. Книги — адмін-панель

Розширені поля та логіка збереження для повноцінного ведення каталогу.

ВиправленняАдмінка

Автори та звʼязок author_book

  • У формі книги — поле «Автори» (мультивибір) замість одного автора.
  • Перший обраний автор — основний (author_id).
  • Таблиця author_book автоматично заповнюється при збереженні.
  • Команда для існуючих записів: php artisan books:sync-authors-pivot.

4. Імпорт книг через Excel

Виправлення

Роки видання

Поля UKR_publish_year та first_publish_year тепер зберігаються незалежно. Раніше при помилці розпізнавання другого поля обидва отримували значення року укр. видання. Також покращено читання років з Excel (числа, дати).

Прибрано з імпорту (на прохання замовника): ISBN, видавництво, мова видання, рейтинг, кількість відгуків — ці поля не перезаписуються при імпорті.

5. Автори

Сайт

Алфавітний фільтр

  • Відображається повний український алфавіт, включно з Є, Ґ, І, Ї.
  • Активні літери — ті, для яких є автори; решта показані приглушено.
  • Фільтр враховує last_name_ua та last_name.
  • Автори з латинським прізвищем (наприклад, Eugenides) відображаються під літерою E, не Є.
Імпорт

Кілька авторів (співавторство)

У Excel: Олександр; Марія в first_name_author та відповідні прізвища в last_name_author, розділені ;.

6. Категорії та підкатегорії

Повноцінна ієрархія категорій у базі, адмінці, імпорті та на публічному сайті.

НовеАдмінка

Адмін-панель

  • Поле «Батьківська категорія» при створенні/редагуванні.
  • У списку категорій — батько, кількість підкатегорій і книг.
  • У формі книги категорії з відступом для підкатегорій.
Імпорт

Підкатегорії в Excel

Шлях через слеш або знак «більше»:

  • Художня / Фантастика
  • Художня > Детектив
  • Кілька категорій: Художня / Фантастика; Нон-фікшн

Книга привʼязується до листової (останньої) категорії в ланцюжку. Відсутні рівні створюються автоматично.

Сайт

Сторінка /categories

  • Картки всіх категорій з превʼю перших 4 обкладинок.
  • Показ батьківської категорії для підкатегорій.
  • Лічильник книг з урахуванням підкатегорій.
  • Сторінка категорії: хлібні крошки, чипи підкатегорій, список книг.

7. Каталог книг на сайті

Сайт

Фільтр категорій

  • Замість довгого списку в сайдбарі — кнопка «Обрати категорії».
  • Модальне вікно з деревом категорій, пошуком і мультивибором.
  • Вибір батьківської категорії включає книги з усіх підкатегорій.
  • Пункт меню «Категорії» у головній навігації.
Покращення

Картка книги

Довгі назви коректно обрізаються (до 2 рядків), пріоритет відображення української назви.

8. Імпорт Gutenberg (фоновий)

Нове

Оптимізація

  • Кешування відповідей API Gutenberg.
  • Відновлення з місця зупинки (--resume).
  • Сповіщення в Telegram після імпорту (за наявності токена).
  • Пропуск повторного збагачення авторів, якщо дані вже є.
Для коректної роботи кешу та стану відновлення на сервері CACHE_STORE не повинен бути array.

9. Відновлення пароля

Виправлення

Поведінка системи

  • При збої SMTP користувач бачить зрозуміле повідомлення замість сторінки помилки 500.
  • Помилка фіксується в логах сервера для адміністратора.

Необхідна дія на сервері: перевірити MAIL_HOST, MAIL_USERNAME, MAIL_PASSWORD, MAIL_FROM_ADDRESS у файлі .env, потім php artisan config:clear.

10. Розгортання змін на сервері

Типова послідовність після завантаження коду на production (foxybooks.club).

Кроки

  1. git pull — отримати останні зміни
  2. composer install --no-dev --optimize-autoloader — за потреби
  3. php artisan migrate — застосувати міграції БД
  4. php artisan books:sync-authors-pivot — один раз, для старих книг без звʼязку author_book
  5. php artisan optimize:clear — очистити кеш
  6. Перевірити права на public/images/placeholders/ (запис для www-data)
  7. Перевірити MAIL_* у .env
Документ на сайті: цей файл доступний за адресою
